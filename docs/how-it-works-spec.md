# How It Works — спецификация страницы

## URL и навигация
- Маршрут: `/how-it-works`
- Заголовок в Header: «Как это работает»

## Механика: CSS Scroll Snap
- 1 экран = 1 карточка (100svh - header)
- Блочный скролл: scroll-snap-type: y mandatory, scroll-snap-stop: always
- Нативный overflow (не Lenis) — стабильный snap на всех устройствах
- StoryRail: точки 1–7 слева, клик — scrollIntoView к карточке

## Структура карточки: боль → решение → эффект

| # | Боль | Решение | Эффект |
|---|------|---------|--------|
| 1 | Заявки теряются | ИИ фиксирует каждый запрос | 0 потерянных лидов |
| 2 | Менеджеры тратят часы на уточнения | ИИ задаёт 2–3 вопроса | Минус 70% ручных вопросов |
| 3 | Нет скорости ответа | Авто-квалификация, горячие — менеджеру | Ответ ≤ 5 минут |
| 4 | Ручной перенос в CRM | Lead → Bot → CRM автоматически | Авто-интеграции 24/7 |
| 5 | Долго собирать прототип | Код → прототип → демо | MVP за 24–72 часа |
| 6 | Нет контроля | Дашборд, отчёты, метрики | Рост пропускной способности |
| 7 | Нужно под ключ | Handoff + инструкции, 24/7 | Под ключ с поддержкой |
