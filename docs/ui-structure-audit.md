# UI Structure Audit — “карточки в карточках” и шаблонность

Цель: убрать визуальную усталость от одинаковых рамок/плиток и разнообразить паттерны секций.

---

## Home: текущие секции и где “перегруз”

### 1) Hero
- **Компонент**: `components/sections/HeroScene.tsx`
- **Layout**: split (текст + граф), FX слой + панель‑карточка с текстом
- **Риск перегруза**: внутри hero уже есть “карта” с border/bg + буллеты → дальше по странице много похожих бордерных блоков

### 2) “Цифры, которые дают уверенность”
- **Компонент**: `components/sections/Proof.tsx`
- **Layout**: grid 2x2/4 карточки, каждая: `rounded-xl + border + elevated bg + padding`
- **Что перегружает**: маленькие карточки, одинаковые рамки, цифры не доминируют → не “WOW”, выглядит как ещё одна секция карточек

### 3) “Что вы получите”
- **Компонент**: `components/sections/PainSolution.tsx`
- **Layout**: сначала grid 3 карточки (FLOW), затем ещё блоки‑карточки (“Без/С”, typewriter, artifacts‑лист)
- **Что перегружает**: несколько разных блоков, но большинство — border/elevated карточки → ощущение “cards everywhere”

### 4) “AI leverage”
- **Компонент**: `components/sections/Leverage.tsx`
- **Layout**: grid 3 одинаковых карточки (border/elevated)
- **Что перегружает**: повторяет паттерн “карточки 3‑в‑ряд” (как кейсы/услуги), визуально шаблонно

### 5) “Сервисы и направления”
- **Компонент**: `components/sections/Tasks.tsx`
- **Layout**: grid 4 карточки‑ссылки (border/elevated)
- **Что перегружает**: ещё один grid с тем же видом (border/padding/hover), мало отличий между секциями

### 6) “Услуги и пакеты”
- **Компонент**: `components/sections/Products.tsx`
- **Layout**: grid 4 `SpecularCard` (карточки со спекулярным FX)
- **Что перегружает**: визуально тоже “grid карточек”, плюс внутри hover‑reveal (внимание/шум)

### 7) “Кейсы”
- **Компонент**: `components/sections/Cases.tsx`
- **Layout**: grid 3 `SpecularCard`
- **Что перегружает**: третий/четвёртый подряд “grid карточек”

### 8) “Как работаем”
- **Компонент**: `components/sections/ProcessPanel.tsx`
- **Layout**: **панель + активный пункт** (отличный не‑card‑grid паттерн)
- **Риск перегруза**: внутри есть одна большая панель с border (норм), но важно не ставить рядом ещё 2 grid‑секции подряд

### 9) “Интеграции и стек”
- **Компонент**: `components/sections/Integrations.tsx`
- **Layout**: split (граф + набор карточек‑блоков)
- **Что перегружает**: справа 4 маленьких блока с border/elevated — снова плитка

### 10) “Почему мы”
- **Компонент**: `components/sections/WhyUs.tsx`
- **Layout**: grid 2 колонки карточек‑фактов (border/elevated)
- **Что перегружает**: одинаковые “факт‑карточки” после предыдущих секций

### 11) FAQ
- **Компонент**: `components/sections/FAQ.tsx`
- **Layout**: список карточек Q/A (border/elevated)
- **Что перегружает**: снова “карточки”, хотя по смыслу можно сделать проще (аккордеон/список без тяжёлых рамок)

---

## Вывод (что будем менять)

- **Numbers**: заменить на WOW‑блок (крупные цифры + count‑up + микро‑графика), убрать “4 одинаковые карточки”.
- **AI Leverage**: заменить grid из 3 карточек на **одну большую панель** с вкладками/сегментами.
- **Services**: оставить карточки, но упростить (меньше рамок) и сделать “живее” (иконка + timing chip + лёгкий gradient hover).
- **Системно**: добавить `SectionShell` (plain/panel/grid) и не допускать >2 grid‑секций подряд.
