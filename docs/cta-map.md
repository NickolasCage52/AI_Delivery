# CTA Map — AI Delivery (Home + ключевые страницы)

Цель: меньше одинаковых кнопок, больше “ритма” и вовлечения.  
Primary (везде): **«Получить план внедрения»**. Secondary: **«Смотреть кейсы»**.

---

## Home (`/`)

1) **Hero**
- **Цель**: быстрый захват заявки / открытие модалки
- **Тип**: primary button + secondary button
- **Primary**: «Получить план внедрения» (modal)
- **Secondary**: «Смотреть кейсы» (scroll to cases)

2) **Micro‑CTA (без больших кнопок)**
- **Proof**: текстовая ссылка «Смотреть кейсы →»
- **Что вы получите**: текстовые ссылки «Показать примеры сценариев →», «Смотреть кейсы →»
- **Интеграции и стек**: текстовые ссылки «Смотреть стек и интеграции →», «Смотреть кейсы →»

3) **Process (после объяснения этапов)**
- **Цель**: конверсия в заявку после “понимания процесса”
- **Тип**: CTA‑блок (`SectionCTA`)
- **Primary**: «Получить план внедрения» (modal)
- **Secondary**: «Смотреть кейсы», «Показать примеры сценариев»

4) **Sticky CTA bar**
- **Цель**: вернуть к действию после первого/второго экрана
- **Тип**: sticky bar (primary button + secondary link)
- **Появление**: после ~1–2 экранов (через `IntersectionObserver`, без scroll listeners)

5) **Final CTA (форма)**
- **Цель**: заявка с контекстом
- **Тип**: мощный финальный CTA + форма
- **Primary**: submit «Получить план внедрения»
- **Secondary**: нет (чтобы не размывать действие)

---

## Cases (`/cases`, `/cases/[slug]`)

- **Hero**: `HeroCTA` (modal) + ссылка на услуги
- **Низ страницы**: `SectionCTA` (можно использовать как “после доказательств”)

---

## Services (`/services`)

- **Hero**: `HeroCTA` (modal) + «Смотреть кейсы»
- **Секции**: `SectionCTA` точечно (после пакетов/направлений), без дубля в каждом блоке
